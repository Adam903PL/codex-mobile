# Historyjka uzytkownika: terminal projektu w aplikacji mobilnej

## User Story

Jako uzytkownik chce uruchamiac komendy terminala dla wybranego projektu z telefonu, aby szybko sprawdzic pliki, uruchomic skrypty albo zobaczyc output bez siedzenia przy komputerze.

## Kryteria akceptacji (AC)

- AC1: Uzytkownik moze otworzyc terminal dla wybranego workspace.
- AC2: Terminal pokazuje informacje o aktualnym projekcie i statusie polaczenia.
- AC3: Uzytkownik moze wyslac komende z aplikacji mobilnej.
- AC4: CLI wykonuje komende lokalnie w katalogu przypisanym do projektu.
- AC5: Output komendy wraca do aplikacji mobilnej i jest widoczny w terminalu.
- AC6: Uzytkownik moze wyczyscic terminal albo przerwac dzialanie, jezeli komenda trwa zbyt dlugo.
- AC7: Jezeli CLI jest offline, aplikacja pokazuje komunikat zamiast udawac, ze komenda sie wykonuje.

## Powod biznesowy

Terminal zwieksza praktyczna uzytecznosc DevLinka, bo pozwala kontrolowac lokalne srodowisko developerskie z poziomu telefonu.
